# Шаблони. Идея, особености, използване. Шаблони на функции

## Задача 1 → Шаблонна библиотека за обработка на хетерогенни масиви

Да се дефинира шаблонна структура `Buffer<T>`, която да служи за работа с масиви от произволен тип. Структурата трябва да съдържа указател към данните `T* data`, текущ брой елементи `int size` и максимален капацитет `int capacity`.

Реализирайте набор от шаблонни функции за управление на този буфер:

- `template <typename T> void init(Buffer<T>& buf, int cap)` → заделя динамична памет за масива
- `template <typename T> void destroy(Buffer<T>& buf)` → освобождава заделената памет
- `template <typename T> bool add(Buffer<T>& buf, const T& element)` → добавя елемент в края (ако капацитетът е достигнат, функцията връща `false`)
- `template <typename T> int find(const Buffer<T>& buf, const T& target)` → връща индекса на първото срещане на `target` или -1
- `template <typename T> T* getAt(Buffer<T>& buf, int index)` → връща указател към елемент на дадена позиция
- `template <typename T> T* findMax(const Buffer<T>& buf, bool (*isGreater)(const T&, const T&))` → намира най-големия елемент в буфера, използвайки подадена функция за сравнение

Реализирайте шаблонни функции за синхронизация с двоичен файл:

- `template <typename T> bool saveToDisk(const Buffer<T>& buf, const char* filename)` → записва съдържанието на буфера в двоичен файл
- `template <typename T> bool loadFromDisk(Buffer<T>& buf, const char* filename, int limit)` → чете до `limit` елемента от двоичен файл директно в буфера
- `template <typename T> bool loadSpecific(const char* filename, T& dest, int index)` → прочита само **един** конкретен запис от файла на позиция `index` и го записва в `dest`

Демонстрирайте работата на шаблоните с два различни типа:

- Буфер от `double`, който съхранява температурни измервания
- Буфер от структура `InventoryItem { int id; double weight; }`, която описва наличност в склад

Програмата трябва да запълни буферите, да намери максималния елемент във всеки от тях, да ги запише в отделни двоични файлове и накрая да прочете произволен запис от файла, за да докаже, че прекият достъп работи коректно.

## Задача 2 → Шаблонна система за изчисляване на сложни изрази

Да се дефинира шаблонна структура `Expression<T>`, която представя математическа операция върху масив от данни.

Структурата трябва да съдържа:

- Указател към масив от данни `T* operands`
- Брой елементи в масива `int count`
- Указател към функция `T (*operation)(const T*, int)`, която дефинира как се пресмята резултатът от тези данни

Реализирайте следните шаблонни функции за работа с изрази:

- `template <typename T> void initExpr(Expression<T>& expr, int size, T (*op)(const T*, int))` → заделя динамична памет за операндите и задава операцията
- `template <typename T> T calculate(const Expression<T>& expr)` → извиква вградената операция върху операндите и връща крайния резултат
- `template <typename T> void fillOperands(Expression<T>& expr)` → чете `count` на брой стойности от конзолата и ги попълва в израза

Реализирайте конкретни функции, които да се подават като параметри на шаблона:

- `getAverage` → намира средно аритметичното на елементите в масива
- `getMax` → намира максималния елемент
- `getSum` → намира сумата на елементите
- `getProduct` → намира произведението на елементите

Реализирайте функция `template <typename T> void logResult(const Expression<T>& expr, const char* filename)`, която:

- Изчислява стойността на израза
- Записва в текстов файл оригиналните операнди и крайния резултат в подходящ формат

Демонстрирайте работата на програмата по следния начин:

- Създайте `Expression<int>` за пресмятане на сума
- Създайте `Expression<double>` за пресмятане на средно аритметично
- Покажете как един и същ шаблон `Expression<T>` може да промени поведението си само чрез подаване на различна функция
