# Двоични файлове. Работа при пряк достъп

## 1. Защо `read()` и `write()` вместо `>>` и `<<`?

В текстовите файлове операторите `<<` и `>>` извършват **форматиране**. Например, числото `int x = 12345;` се превръща в петте символа `'1'`, `'2'`, `'3'`, `'4'` и `'5'`.

В двоичните файлове ние искаме да запишем **директно съдържанието на паметта**. За целта използваме два специфични метода:

- `write(const char* buffer, std::streamsize size)` → копира `size` на брой байта от адреса `buffer` директно във файла
- `read(char* buffer, std::streamsize size)` → копира `size` на брой байта от файла директно в адреса `buffer`

### Защо ни е `reinterpret_cast<const char*>`?

Тъй като методите `read()` и `write()` очакват указател от тип `char*` (защото `char` е точно 1 байт), трябва да кажем на компилатора да интерпретира адреса на нашия обект като поредица от байтове.

```cpp
int number = 12345;

file.write(reinterpret_cast<const char*>(&number), sizeof(number));
```

## 2. Работа със структури

Най-голямата сила на двоичните файлове е възможността да запишем цяла структура с една команда.

```cpp
struct Student {
    int id;
    
    double grade;
    
    // Важно: фиксирана дължина
    char name[32];
};

Student s{ 1, 5.50, "Ivan" };

// Запис на целия обект наведнъж
file.write(reinterpret_cast<const char*>(&s), sizeof(Student));
```

## 3. Пряк достъп

Както споменахме предния път, прекият достъп е възможен само при фиксирана дължина на записа.

Ако искаме да прочетем `N`-тия студент:

- **Изчисляваме позицията:** `pos = (N - 1) * sizeof(Student)`
- **Преместваме курсора за четене:** `file.seekg(pos, ios::beg)`
- **Четем обекта:** `file.read(reinterpret_cast<char*>(&s), sizeof(Student))`

## 4. Предимства на двоичния подход

- **Скорост:** Няма нужда от парсване на текст (превръщане на "123" в число); данните отиват директно от диска в променливата
- **Точност:** При текстовите файлове числата с плаваща запетая могат да загубят прецизност при закръгляне; в двоичен вид те се запазват бит по бит
- **Обем:** Числото `1000000000` в текстов файл е 10 байта (10 цифри), в двоичен е винаги 4 байта (`int`)
