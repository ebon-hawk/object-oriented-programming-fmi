# Двоични файлове. Работа при пряк достъп

### Задача 1 → Система за управление на записи

Да се реализира програма за поддръжка на база данни от изделия, съхранявани в двоичен файл. Всеки запис се описва чрез следната структура:

```cpp
struct Product {
    // Описание на изделие
    char descr[28];
    
    // Номер на изделие (ключ)
    int id;
    
    // Цена на изделие
    double price;
};
```

Номерът на изделието (`id`) е цяло число в интервала `[43000, 43099]`. Този номер определя логическата позиция на записа във файла.

Програмата трябва да поддържа следните функционалности:

- **Инициализация:** Създава файл, съдържащ точно 100 "празни" записа от тип `Product`
- **Запълване:** Позволява добавяне на информация в конкретни записи, указани чрез техния номер
- **Извеждане:** Показва информацията за всички налични изделия, сортирана във възходящ ред по техния номер
- **Актуализация:** Променя цената (`price`) на съществуващо изделие по подаден номер
- **Включване:** Добавя нов запис за изделие в рамките на позволения интервал
- **Изтриване:** Премахва съдържанието на съществуващ (непразен) запис от файла

### Задача 2 → Статистика на целочислен масив

Да се напише програма, която записва елементите на масив от цели числа в двоичен файл, след което извежда на екрана компонентите на файла, както и средно аритметичното на елементите му.

### Задача 3 → Конвертиране на сортирани тройки числа

Да се напише програма, която чете от текстов файл редици от тройки цели числа, сортира елементите на всяка от тях във възходящ ред и ги записва в двоичен файл.

На един ред в текстовия файл има 3 числа, разделени с интервал.

### Задача 4 → Решаване на уравнения от двоичен източник

Да се напише програма, която чете от двоичен файл редица от тройки реални числа, представящи коефициенти на квадратни уравнения.

Програмата да намира корените на дефинираните квадратни уравнения и да ги запише в текстов файл - за всяко уравнение на отделен ред. Ако уравнение няма реални корени или не е квадратно да се запише "НРК".

### Задача 5* → *Hex Viewer*

Да се реализира програма за изглед и модификация на двоични файлове. При стартиране на програмата потребителят въвежда път до файл, който се зарежда изцяло в паметта.

Работата с програмата се осъществява чрез прост конзолен интерфейс със следните команди:

- `view` → извежда съдържанието на файла (всеки байт се отпечатва в **шестнадесетична бройна система**, последван от неговата символна интерпретация)
	- Ако байтът съответства на малка или голяма латинска буква, се отпечатва самият символ
	- В противен случай се отпечатва точка (`.`)
- `change <index> <value>` → променя стойността на байта на съответната позиция
- `remove` → премахва последния байт от съдържанието на файла
- `add <value>` → добавя нов байт в края на данните
- `save` → записва направените промени обратно в оригиналния файл
- `save as <filename>` → записва текущото състояние на данните в нов файл с посоченото име

За проверка на функционалността използвайте `myData.dat` файла, генериран чрез следния код:

```cpp
int x = 25409;
std::ofstream file("myData.dat", std::ios::binary);

file.write((const char*)(&x), sizeof(x));
```

Пример (зареждане и визуализация):

```cpp
Enter file path: myData.dat
File loaded. 4 bytes read.

> view
41 63 00 00 | Ac..
```

**Обяснение:** 25409 в шестнадесетичен вид е `0x6341`. Поради *little-endian* подредбата, байтовете са 41 (`'A'`), 63 (`'c'`), следвани от два байта с нули, които се отпечатват като точки.
